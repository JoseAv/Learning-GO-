# Interfaces

## Como contrato

Es un contrato es un conjunto de metodos que un tipo debe de implementar.


``` go

type Hablador interfaces {
    Hablar() string
}

type Perro struct{}

// Si un type satisface a todos los metodos de una interface, entonces automaticamente puede utilizar el tipo Hablado
func (p Perro) Hablar() string{
    return "Wuff!!"
}


func main(){

// Aqui creamos una interfaces de tipo hablador.
//  y la clase que lo cumple es Perro.
var p Hablador = Perro{}
fmt.Println(p.Hablar())
}

```


Al cumplir el contrato estamos usando el polimorfimos para tener muchas formas de una interfaces.

#

``` go
func imprimirCualquierCosa(v interface{}){
    fmt.Println(v)
}


func main(){
    imprimirCualquierCosa(42) // int
    imprimirCualquierCosa("Hola") // string
    imprimirCualquierCosa(3.14) // float64
}

```

Al interface estar vacio, quiere decir que va a aceptar cualquier valor, esto es equivalente a tipo any, pero el tipo any pierde los beneficios de tener algo tipado.
