# Clases

Type tambien es el equivalente para crear metodo + clases pero tienen un sintaxis diferente en js se veria algo asi


***JS:***
``` js
class Celsios{
    constructor(celsio){
        this.celsio = celsio
    }

    toFahrenheit() {
    return this.celsio * 9/5 + 32;
  }

}


const temp = new Celsio(25)
temp.toFahrenheit() // lo convierte

```

***Go:***
``` go
type Celsios float64

// (c Celsios) Parametros creado
// toFahrenheit() float64 -> nombre de la funcion y que devuelve
func (c Celsios) toFahrenheit() float64{
    return (c * 9/5 + 32);
}

// Como creamos el tipo celcios

var Temp = Celsios(25)
// El tipo celsios tiene un metodo -> toFahrenheit()
fmt.Println(toFahrenheit())

```

aunque puede ser algo complicado de ver, es el mismo comportamiento que las clases, nosotros definimos la clase con tipe y ahora para decir que metodo es de esa clase.


Ahora talves puedas pensar que la forma en la que declaramos la funcion es porque c es de tipo Celsios y si es correcto, pero esta es la forma de indicar que este metodo es de tal clase.

## Clases vacias

Pero como definimos un clase vacia es decir en otros lenguajes podemos no pasar nada en el contructor pero en Go es necesario definir el tipo

``` go
type newClass //❌ Esto no existe

type Calculadora struct{} // Esto si se puede

```

pero siempre hay que definir un tipo, aunque no se use, es requerido.

Entonces si el type siempre requiere una tipo exactamente es el metodo asociado

***Ejemplo***


``` go
type Calculadora struct{} // Esto si se puede


// c Calculadora es la forma de indicar este metodo es de este typo creado, aunque no lo usemos, o el type no valga nada, se hace de esta forma
func (c Calculadora) sum() float64{
    return 10+10;
}

```
